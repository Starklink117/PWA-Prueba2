            <h4>¿Qué es una aplicación orientada a servicios?</h4>

            <p>
                Una Aplicación Orientada a Servicios (SOA, por sus siglas en inglés, 
                Service-Oriented Architecture) es un enfoque arquitectónico para diseñar 
                sistemas de software en el que los componentes de software se organizan y 
                comunican entre sí a través de servicios. 
            </p>

            <h4>
                Características 
                <i class="fa-solid fa-list-check me-2" style="color: #4497c1;"></i>
            </h4>

            <ul>
                <li>
                    <b>Descomposición en Servicios: </b>
                    En SOA, una aplicación se divide en servicios independientes y autónomos. 
                    Estos servicios encapsulan funciones específicas y se diseñan para ser 
                    reutilizables en toda la aplicación o incluso en diferentes aplicaciones.
                </li>
                <li>
                    <b>Estándares de Comunicación: </b>
                    SOA utiliza estándares de comunicación, como el protocolo HTTP, XML y SOAP 
                    (Simple Object Access Protocol), para permitir la comunicación entre 
                    servicios de manera uniforme.
                </li>
                <li>
                    <b>Reutilización: </b>
                    Los servicios se diseñan para ser reutilizables en múltiples partes 
                    de una aplicación o incluso en diferentes aplicaciones dentro de la 
                    organización. Esto promueve la eficiencia en el desarrollo de software.
                </li>
                <li>
                    <b>Escalabilidad: </b>
                    La seguridad es una consideración importante en SOA. Los servicios pueden 
                    implementar medidas de seguridad, como autenticación y autorización, 
                    para proteger los datos y garantizar el acceso autorizado.
                </li>
                 <li>
                    <b>Componentes Autónomos: </b>
                    Los servicios son componentes autónomos que pueden evolucionar de 
                    forma independiente sin afectar a otros servicios o a la aplicación 
                    en su conjunto.
                </li>
            </ul>
